Vanilla javaScript

I’ve been actively working with Vanilla JavaScript to deepen my understanding of core web development concepts. Using plain JavaScript without libraries or frameworks allows me to grasp the fundamentals of DOM manipulation, event handling, and asynchronous operations like fetching APIs. It’s helped me build interactive, responsive user interfaces from scratch and debug more efficiently.


React

I’ve been exploring React to build dynamic and efficient user interfaces for modern web applications. React’s component-based architecture allows me to create reusable UI elements that streamline development and improve code maintainability. Through working with hooks, state management, and props, I’m gaining deeper insight into how interactive elements behave and respond to user input.


Node.Js

I’ve been learning Node.js to develop scalable and efficient server-side applications using JavaScript. Its non-blocking, event-driven architecture makes it perfect for handling concurrent requests and building APIs with fast response times. I enjoy using Node.js to manage server logic, interact with databases, and integrate tools like Express.js for streamlined routing and middleware.


MongoDB

I’ve been exploring MongoDB as a powerful NoSQL database solution for modern web applications. Its flexible, document-based structure lets me store data in JSON-like formats, which feels natural and intuitive—especially when working with JavaScript frameworks like Node.js. I appreciate how MongoDB allows dynamic schemas, making it easier to iterate quickly during development.
